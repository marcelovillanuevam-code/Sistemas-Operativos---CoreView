CoreView

Simulador interactivo de algoritmos de CPU scheduling y page replacement para sistemas operativos.

①

Define tus procesos

Configura tiempos de llegada, CPU burst, prioridad y memoria. Importa desde un archivo .txt o ingresa manualmente.

②

Visualiza algoritmos

Observa paso a paso cómo FCFS, SJF, RR, MLFQ y otros toman decisiones. Diagrama de Gantt, cola de listos y estados.

③

Compara y analiza

Métricas de rendimiento (TAT, WT, RT) y page replacement (FIFO, LRU, Optimal) lado a lado para encontrar el mejor.

¿Qué algoritmos incluye?

Planificación de CPU

  • FCFS — First Come First Served
  • SJF — Shortest Job First
  • SRTF — Shortest Remaining Time First
  • HRRN — Highest Response Ratio Next
  • RR — Round Robin
  • Priority — Por prioridades (expropiativo)
  • MLQ — Multilevel Queue
  • MLFQ — Multilevel Feedback Queue

Page Replacement

  • FIFO — First In First Out
  • LRU — Least Recently Used
  • Optimal — Algoritmo óptimo (referencia teórica)
  • Clock — Algoritmo del reloj
  • Second Chance — Segunda oportunidad

Flujo recomendado

  1. Entrada — Define procesos (PID, llegada, CPU burst, prioridad, pages) y configura la memoria.
  2. Scheduling — Elige un algoritmo de CPU scheduling y observa la animación.
  3. Memoria — Revisa la asignación inicial de frames físicos.
  4. Page Replacement — Ejecuta un algoritmo sobre la reference string.
  5. Métricas / Comparison — Analiza resultados y compara algoritmos.

Glosario de términos

Referencia rápida de la terminología utilizada en el simulador.

Métricas de Scheduling

CT — Completion Time (Tiempo de finalización)
Instante en el que un proceso/thread termina su ejecución.
TAT — Turnaround Time (Tiempo de retorno)
Tiempo total que pasa el proceso en el sistema. TAT = CT − Arrival. Menor es mejor.
WT — Waiting Time (Tiempo de espera)
Tiempo total que el proceso pasó en la cola de listos. WT = TAT − Burst. Menor es mejor.
RT — Response Time (Tiempo de respuesta)
Tiempo desde la llegada hasta que recibe CPU por primera vez. RT = First Run − Arrival. Menor es mejor.
CPU Utilization
Porcentaje del tiempo total en que la CPU estuvo ejecutando trabajo (no inactiva).
Throughput
Threads completados por unidad de tiempo.
Context Switch
Cambio de contexto: cuando la CPU pasa de ejecutar un proceso/thread a otro.

Algoritmos de Scheduling

FCFS — First Come First Served
No expropiativo. Atiende en orden de llegada. Simple pero penaliza procesos cortos detrás de largos.
SJF — Shortest Job First
No expropiativo. Selecciona el proceso con menor CPU burst. Optimiza WT pero requiere conocer la duración a priori.
SRTF — Shortest Remaining Time First
Versión expropiativa de SJF. Si llega un proceso con menor tiempo restante, expropia al actual.
HRRN — Highest Response Ratio Next
No expropiativo. Selecciona por ratio (espera + burst) / burst. Evita inanición.
RR — Round Robin
Expropiativo. Cada proceso recibe un quantum fijo de CPU. Justo para todos.
Priority — Por prioridades (expropiativo)
Selecciona el de mayor prioridad (menor número). Puede causar inanición sin envejecimiento.
MLQ — Multilevel Queue
Múltiples colas con prioridad fija. Cada cola usa su propio algoritmo.
MLFQ — Multilevel Feedback Queue
Como MLQ pero los procesos pueden cambiar de cola dinámicamente. Soporta envejecimiento.

Memoria y Page Replacement

Frame (Marco)
Bloque físico fijo de memoria. La memoria se divide en frames del mismo tamaño que las pages.
Page (Página)
Bloque lógico de un proceso. Cada page puede cargarse en cualquier frame libre.
Page Fault (Fallo de página)
Ocurre cuando la page solicitada no está en memoria. Hay que cargarla, posiblemente reemplazando otra.
Hit / Hit Rate
Acierto: la page ya estaba en memoria. El hit rate es hits / total.
Internal Fragmentation
Espacio desperdiciado dentro del último frame asignado a un proceso cuando éste no llena el frame completo.
FIFO / LRU / Optimal / Clock
Estrategias para decidir qué page expulsar cuando ocurre un page fault: la más vieja, la menos recientemente usada, la que se usará más tarde, o usando un bit de referencia circular.
